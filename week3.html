Kaplan-Meier (Product-Limit) Estimator (Section 4.2)

For survival data one commonly wants to find an estimate of the survival function \(S(t)\). The standard estimator of \(S(t)\) is the Kaplan-Meier estimator. Assuming \(t_1 < t_2 < \ldots, t_D\) for \(D\) distinct times, the Kaplan-Meier estimate is defined as follows,

\[ \hat{S}(t) = \prod_{t_i \leq t} \left[1- \frac{d_i}{Y_i} \right], \]

where at \(t_i\) there are \(d_i\) events (deaths) and \(Y_i\) number of individuals who are at risk at time \(t_i\). To find the Kaplan-Meier estimates in R, we can use the survfit( ) function from the survival library.

data(drug6mp)
fit <- survfit(Surv(t2, relapse) ~ 1, data = drug6mp)
summary(fit) #Table 4.1a

Confidence Intervals (Section 4.3)

\((1-\alpha)\times100\%\) confidence intervals for the product-limit estimates can automatically be constructed through survfit. The arguments are conf.type for the type of interval to be constructed (default is log, but others include log-log, plain [linear], `none) and conf.int for the length of the interval (default is .95). (Note: The log option is R is different than the log-transformed intervals in Klein and Moeschberger, which corresponds to conf.type = log-log. )

Nelson-Aalen Estimator

Recall the following relationship, \[ H(t) = - \log S(t). \] Thus, we can estimate the cumulative hazard from the Kaplan-Meier estimate, \[\hat{H}(t) = - \log \hat{S}(t).\] We shall return to the drug6mp dataset for illustration.

H.km <- -log(fit$surv); H.km

An alternate method of estimating the cumulative hazard is through the Nelson-Aalen estimate, \[\tilde{H}(t) = \sum_{t_i \leq t} \frac{d_i}{Y_i}.\] Both \(d_i\) and \(Y_i\) can be extracted from survfit.

#Nelson-Aalen Estimate of H(t):
h               <- fit$n.event/fit$n.risk
H.na            <- cumsum(h) #Nelson-Aalen estimates
s               <- fit$n.event/fit$n.risk^2
V.na            <- cumsum(s) #Variance estimate for the N-A estimator
NAest           <- cbind(H.na, sqrt(V.na))
colnames(NAest) <- c("NA-Est.","Std. Err")

Likelihood Ratio Confidence Intervals

As disucssed in class one may use a likelihood ratio type confidence interval. To generate such confidence intervals one will use the LRci function on GitHub. A survfit function is required as well as the time, t, of interest. A root finder is used to find the upper and lower limits of the confidence interval. Since the upper root is searched within the interval \((0, \infty)\), we set the upper limit to be an arbitrarily high number, 100. However, the root may not lie in this interval and a warning will appear. In such cases, one may change the upper.lim value and run the function again.

Using the ALL group for the bmt dataset and being interested in \(S(365)\) we will try to find a \(95\%\) confidence interval using the likelihood ratio approach.

Confidence Bands (Section 4.4)

Confidence bands are different from pointwise confidence intervals. Confidence intervals are only valid for a single fixed time point; whereas confidence bands are valid for the entire survival function. Probabilistically, we want to find two random functions \(L(t)\) and \(U(t)\) such that \[1-\alpha = \Pr(L(t) \leq S(t) \leq U(t)) \hspace{.2in} \forall t_L \leq t \leq t_U. \]

There are two popular types of confidence bands: Hall-Wellner and Equal Probability. The survival library does not, to the best of my knowledge, have a function or option to include confidence bands. So we will install and load another package, km.ci.

#install.packages('km.ci')
library(km.ci)

A quick Google search will yield multiple different functions and/or libraries to produce confidence bands for survival data. My personal favorite is http://www.ddiez.com/teac/surv/conf-bands.R. However, R markdown will not behave nicely with this function but I encourage you all to look at it!

Using km.ci() we can find confidence bands for our bmt example in Example 4.2. The function is rather limited and the values are slightly off compared to the tables in the book. SAS has a more in-depth selection for confidence bands (and intervals). However, the values outputted from SAS are also slightly off.

data(bmt)

my.surv <- survfit(Surv(bmt$t2[bmt$group==1], bmt$d3[bmt$group==1])~1, conf.type = 'none')

ep.band <- km.ci(my.surv, tl = 86, tu = 609, method = 'logep')
hw.band <- km.ci(my.surv, tl = 86, tu = 609, method = 'loghall')

#Plotting them
plot(my.surv, xlim = c(100, 600), main="Kaplan-Meier Estimate with 95% confidence bands \n Figure 4.5 (pg. 112)",
     xlab="Time (t)", ylab="Estimated Survival Function")
